# 365-Day Prepaid SIM Comparison (Australia)

A comparison of every 365-day (and near-365-day) prepaid SIM plan available in Australia — 15 providers, 30 plan tiers — cross-checked against real ProductReview.com.au and Trustpilot ratings, with a sortable table (desktop headers / mobile dropdown), a mobile card layout, and dark/light theming.

No build step, no dependencies, no framework — plain HTML/CSS/JS:

```
index.html      markup
styles.css       all styling, incl. @font-face declarations
script.js        table sort + mobile card behaviour
fonts/           Big Shoulders (display) + Public Sans (body), self-hosted .ttf
logos/           15 provider icons, .png
```

Everything is a real file referenced by relative path — nothing is inlined as base64, so any file can be swapped independently.

## Deploy on GitHub Pages

1. Push this folder's contents to a new repo (or a `docs/` folder / `gh-pages` branch of an existing one), keeping the file layout above intact.
2. In the repo, go to **Settings → Pages**.
3. Under **Build and deployment**, set **Source** to "Deploy from a branch," pick the branch (and `/docs` folder if that's where you put it), and save.
4. GitHub will publish it at `https://<username>.github.io/<repo-name>/` within a minute or two.

That's it — no npm install, no build command, nothing else to configure.

## Keeping the logos fresh

`.github/workflows/check-logo-freshness.yml` runs quarterly (1 Jan / 1 Apr / 1 Jul / 1 Oct, plus manually via the Actions tab → "Run workflow") and:

1. Re-fetches each provider's icon from Google's favicon service.
2. Compares it against the checked-in file in `logos/`.
3. If anything differs, opens a PR (`chore/logo-refresh`) with the updated file(s) — it never pushes straight to `main`.

**Heads up on the first run:** five of the fifteen logos (Kogan, Boost, Telstra, Telsim, Vodafone) were originally sourced by hand at higher resolution than Google's favicon service returns, so the very first scheduled run will likely open a PR "changing" those back down to the Google version. That's expected, not a sign anything broke — just close the PR without merging if you'd rather keep the higher-res originals, or merge it to standardize all 15 logos on the same source going forward.

## Updating the data later

The content was generated by hand from a research pass in July 2026. Since there's no build tooling, edits are direct:

- **Plan data / prices / ratings** — `index.html`, inside `<table id="plan-table">`.
- **Copy / recommendations / footer sources** — the relevant `<section>` in `index.html`.
- **Colors, spacing, layout, the mobile breakpoint** — `styles.css`.
- **Sort behaviour** — `script.js`.
- **Swapping a provider's icon by hand** — replace the matching file in `logos/` (keep the same filename, or update the `<img src>` in `index.html` if you rename it).
