# 365-Day Prepaid SIM Comparison (Australia)

A comparison of every 365-day (and near-365-day) prepaid SIM plan available in Australia — 15 providers, 28 plan tiers, all in one sortable table — cross-checked against real ProductReview.com.au and Trustpilot ratings, not vendor marketing. Sortable on desktop (click a column header), a dropdown-driven equivalent on mobile, a stacked card layout below 860px, and dark/light theming.

Where a provider's review sample is too thin to trust (e.g. 2 ProductReview reviews) or a listing genuinely doesn't exist, that's shown as a greyed-out pill rather than a colored score or a silent omission — the aim is an honest picture, not a tidy one.

Plain HTML/CSS/JS, no build step, no framework:

```
index.html                    markup
styles.css                     all styling, incl. @font-face declarations
script.js                      table sort + mobile card behaviour
fonts/                         Big Shoulders (display) + Public Sans (body), self-hosted .ttf
images/logos/                  15 provider icons, .png
images/icon.png                site favicon
productreview-snapshot.json    last-known ProductReview ratings, read/written by the monthly check
```

Everything is a real file referenced by relative path — nothing is inlined as base64, so any file can be swapped independently.

## Automation

Three scheduled GitHub Actions workflows, each doing one narrow, verifiable thing:

- **`check-logo-freshness.yml`** — monthly (1st of the month, plus manual runs from the Actions tab). Re-fetches each provider's icon from Google's favicon service, and if the hash differs from what's checked into `images/logos/`, commits and pushes straight to `main` — no PR, no separate branch. The whole check is a single Python script inlined in the workflow file, no separate `scripts/` folder.

  **Heads-up:** five logos (Kogan, Boost, Telstra, Telsim, Vodafone) were originally sourced by hand at higher resolution than Google's favicon service returns, so the first run overwrote those with the lower-res version. Edit the `DOMAINS` dict in the workflow if you'd rather it skip those five.

- **`check-rating-freshness.yml`** — monthly, offset an hour after the logo check. Re-fetches ProductReview's `aggregateRating` JSON-LD for all rated providers, diffs against `productreview-snapshot.json`, commits the snapshot if anything moved, and opens a GitHub issue listing exactly what changed. It does *not* touch `index.html` directly — a rating change should also prompt a look at the pill's color threshold and the note text next to it, not just a number swap. Trustpilot isn't checked here: it 403s on plain HTTP requests (confirmed by testing, not assumed), so it stays manual.

- **`annual-content-review.yml`** — once a year (1 July). Opens a GitHub issue assigned to the repo owner with a checklist: re-verify every provider's plan still exists, check for new market entrants, re-pull Trustpilot ratings by hand, and re-check the "ruled out" list (Belong, Circles.Life, Moose Mobile, SpinTel, Gomo, Felix Mobile, Yomojo, Bendigo Telco). GitHub's own notification system emails the assignee — no SMTP credentials, no secrets, nothing to configure.

## Editing the data

Generated by hand from a research pass in July 2026, refreshed since. No build tooling, so edits are direct:

- **Plan data / prices / ratings** — `index.html`, inside `<table id="plan-table">`.
- **Copy / recommendations / footer sources** — the relevant `<section>` in `index.html`.
- **Colors, spacing, layout, the mobile breakpoint** — `styles.css`.
- **Sort behaviour** — `script.js`.
- **A provider's icon** — replace the matching file in `images/logos/` (keep the filename, or update the `<img src>` in `index.html` if you rename it).
