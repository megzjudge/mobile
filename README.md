# 365-Day Prepaid SIM Comparison (Australia)

A comparison of every 365-day (and near-365-day) prepaid SIM plan available in Australia — 15 providers, 30 plan tiers — cross-checked against real ProductReview.com.au and Trustpilot ratings, not vendor marketing. Sortable table on desktop (click a column header), a dropdown-driven equivalent on mobile, a stacked card layout below 860px, and dark/light theming.

Plain HTML/CSS/JS, no build step, no framework:

```
index.html      markup
styles.css       all styling, incl. @font-face declarations
script.js        table sort + mobile card behaviour
fonts/           Big Shoulders (display) + Public Sans (body), self-hosted .ttf
logos/           15 provider icons, .png
```

Everything is a real file referenced by relative path — nothing is inlined as base64, so any file can be swapped independently.

## Logo freshness

`.github/workflows/check-logo-freshness.yml` runs quarterly (1 Jan / 1 Apr / 1 Jul / 1 Oct, plus manually via the Actions tab) and:

1. Re-fetches each provider's icon from Google's favicon service.
2. Compares it against the checked-in file in `logos/`.
3. If anything differs, commits and pushes straight to `main` — no PR, no separate branch.

The whole check is a single Python script inlined in the workflow file, no separate `scripts/` folder.

**First run heads-up:** five logos (Kogan, Boost, Telstra, Telsim, Vodafone) were originally sourced by hand at higher resolution than Google's favicon service returns, so the first scheduled run will likely overwrite those with the lower-res version. Expected, not a bug — edit the `DOMAINS` dict in the workflow if you'd rather it skip those five.

## Editing the data

Generated by hand from a research pass in July 2026, no build tooling, so edits are direct:

- **Plan data / prices / ratings** — `index.html`, inside `<table id="plan-table">`.
- **Copy / recommendations / footer sources** — the relevant `<section>` in `index.html`.
- **Colors, spacing, layout, the mobile breakpoint** — `styles.css`.
- **Sort behaviour** — `script.js`.
- **A provider's icon** — replace the matching file in `logos/` (keep the filename, or update the `<img src>` in `index.html` if you rename it).
